  STEP_1 BEAM_GEOMETRY → STEP_2 SUPPORTS → STEP_3 POINT_LOADS → STEP_4 DISTRIBUTED_LOADS → STEP_5 VALIDATION_CHECK

## Step 1 — Find the exercise diagram
Student pages often contain:
- UPPER region: the actual exercise (beam line, supports, load arrows, dimension ticks) — THIS is your source.
- LOWER region: scratch work, equilibrium equations, answers — IGNORE unless the diagram is missing.
Set image_focus.exercise_region accordingly and describe what you ignored in image_focus.ignored_regions.

## Step 2 — Build the beam model (x-axis along beam, in meters)
1) Rotate mentally so the beam is horizontal left→right (even if the photo shows it vertical).
2) Find the dimension line with tick marks and numbers (e.g. 4.2, 4, 4, 2, 2) — this is the ground truth for distances.
3) Mark ALL labeled points in order (e.g. C, A, D, E, F, B) and assign x by cumulative segment sum from the left end.
4) L = sum of ALL segment lengths = x of the rightmost beam end. Do NOT default to L=6 or assume supports at 0 and L.
5) Pin/roller may be at interior points (e.g. pin at A, x=4.2 — not x=0). Set ra_pos/rb_pos from actual support labels.
6) Ignore student equilibrium scribbles (ΣFx=0, Ax+8.66=0) — extract only loads drawn on the beam diagram.

Rules:
- Beam units ONLY: meters, ton (force), ton/m (distributed magnitude), ton·m (moment). Positive Fy = downward load.
- Distributed loads: distributed_loads[] ONLY (see distributed_loads_protocol) — never type distributed in loads[].
- Separate stacked loads: point/inclined label at arrow tip; same-x loads = separate JSON entries.
- Load x: project vertically from beam contact point (inclined = tip) to dimension line — never from label text position alone.
- key_points_m: sorted unique x values for beam ends, supports, and every load boundary/application point.
- segments: consecutive intervals along the beam; length_m should match to_x - from_x.
- supports: at least one entry for cantilever (fixed at x=0); two entries (pin + roller) for simply supported unless drawn otherwise.
- SUPPORT TYPE CLASSIFICATION (critical — no left/right assumption): classify EACH support
  independently by its own drawn symbol, by COUNTING the marks — never by whether it is the left
  or right support, and never by habit/default. MULTIPLE repeated short diagonal tick-marks
  (hatching, ////) coming out of the support = "pin" (resists Fx+Fy, two reactions). Circles/wheels
  underneath, OR a SINGLE plain bar/line with no repeated diagonal ticks, OR nothing underneath at
  all = "roller" (resists Fy only, one reaction). A lone straight line/rail is NOT hatching by
  itself — do not classify it as "pin" just because some line is drawn. "fixed" is ONLY for a
  cantilever wall-embedment — never assign "fixed" to a support inside a two-support beam.
- hatch_count / has_circles_or_wheels (mandatory per support, two-support beams): before deciding
  "type", literally COUNT the short diagonal tick strokes you can see under that specific support
  and report the integer in hatch_count (0 if none, e.g. a single plain bar or nothing). Set
  has_circles_or_wheels=true only if circles/wheels are drawn under that support. This count must
  be independent per support and must NOT default to "left support has more marks" — it is equally
  common for the ROLLER (fewer/no marks) to be on the LEFT and the PIN (hatched) to be on the
  RIGHT. Re-look at each support's own drawing before answering; do not pattern-match to the most
  common textbook layout.
- ra_pos = x of the support whose symbol is "pin"; rb_pos = x of the support whose symbol is
  "roller" — determined by the symbol actually drawn at each one (NOT by assuming A=pin/B=roller,
  and NOT the right beam tip if there is an overhang past B).
- DUAL UNITS on labels: use ton in parentheses, e.g. 20kn/m (2t/m) → w=2; 200kn (20t) horizontal → Fx=20.
- COG units ONLY: meters for positions and custom dimensions; areas in m² in notes. catalog_key one of: IPE, IPN, IPB, UPB, LPN שווה, LPN שונה, צינור עגול, RHS מלבני.
- Never output kN or cm in JSON values (convert kN→ton if written on paper).
- Do NOT assume pin at x=0 and roller at x=L — read actual support positions from labeled points.
- Sanity check: sum(segments.length_m) must equal L; key_points_m must include every labeled point and support.
- Cantilever: fixed at left x=0, free right — set support_mode=cantilever.
- Handwriting: extract all visible numeric labels near arrows, dimensions, and support marks.
- If only one topic is present, set exercise_type accordingly. If unclear, use "unknown" but still fill notes with what you read.
