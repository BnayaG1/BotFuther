LOAD POSITION PROJECTION PROTOCOL (mandatory for EVERY load):

Before assigning x [m] to any load, locate its action point ON THE BEAM AXIS, then project vertically down to the dimension line below.

## Step A — Find the load center on the beam
- **Point load (vertical ↓/↑):** the center is where the arrow meets the beam (middle of application on the beam axis).
- **Inclined load:** the center is EXACTLY where the arrow tip touches / points to the beam — NOT where the magnitude label is written.
- **Moment:** the center is the middle of the curved arrow on the beam axis (where the moment acts).
- **Moment vs roller B:** the curved arrow center is NEVER at the roller symbol — project to the nearest dimension tick (often F, one segment before B). Do NOT set label_at=B for a moment 2m before the beam end.
- **Distributed load:** start_x and end_x = vertical projection of span start/end on beam to dimension line (see distributed_loads_protocol).

## Step B — Vertical drop to dimension line
From that beam point, drop a **vertical line** (perpendicular to the beam) down to the **dimension chain** (tick marks + numbers under the beam).
The x [m] of the load = the x on the dimension line where this vertical meets it (same x as on the beam axis).

## Step C — Match to dimension chain
- If the vertical hits a labeled tick (A, C, D, B…), set label_at to that letter and x from labeled_points.
- If it hits between ticks, x = cumulative sum from the left end to that projection.
- NEVER take x from a number written above the arrow tip if that x differs from the vertical projection to the dimension line.
- NEVER place a load at mid-span by visual guess — always use the vertical projection rule.

## Stacked loads at similar x
Each load gets its OWN projection — moment, inclined, and point at the same station share the same x only if their beam contact points align vertically.
