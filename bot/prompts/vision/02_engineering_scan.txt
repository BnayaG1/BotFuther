## 1. Structural elements
A) Supports — identify EACH symbol INDEPENDENTLY by what is actually drawn under/at it.
   NEVER assume type from left/right position — the left support is NOT always "pin" and the
   right support is NOT always "roller". Classify EACH support on its own visual symbol,
   counting the actual marks drawn — do not guess from habit:
   • MULTIPLE short diagonal parallel tick-marks (hatching, ////, a "half-chain" pattern —
     TWO OR MORE repeated short strokes) drawn coming out of the support = type "pin" = סמך קבוע
     — resists BOTH Fx (axial) and Fy (vertical), i.e. TWO reaction components.
   • A SINGLE straight bar/line under the triangle (with NO repeated diagonal hatch strokes),
     OR circles/wheels drawn under the support, OR a plain triangle with nothing under it at all
     = type "roller" = סמך נייד — resists ONLY Fy (vertical), i.e. ONE reaction component.
     A lone flat line/rail is NOT hatching — do not classify it as "pin" just because a line is
     present; hatching means several repeated short diagonal ticks, not one continuous bar.
   • "fixed" (wall / full hatch embedment, resists Fx, Fy, AND moment) applies ONLY to a
     cantilever exercise (one wall-mounted end, free tip) — NEVER use "fixed" for a support in a
     two-support (simply supported) beam. If a two-support beam has a support that looks
     hatched/embedded, classify it as "pin", not "fixed".
   • It is EQUALLY common for the roller (fewer/no hatch marks) to be drawn on the LEFT and the
     pin (hatched) on the RIGHT as the other way around. Before deciding, literally COUNT the
     diagonal tick strokes under EACH support separately and report that count (hatch_count) —
     do not pattern-match to "left support usually looks like X" from habit or prior exercises.
B) Structure type — classify the whole system:
   • simply_supported — beam on pin + roller (2 supports)
   • cantilever — one fixed end, free tip
   • multi_span — 3 or more vertical supports on one continuous beam line
   • gerber — internal hinges separating spans (Gerber beam)
   • truss — triangulated bars with pin joints (not a continuous beam — note in JSON)
   • frame — rigid joints (note if standard beam solver may not apply)
C) Internal hinges — list every hinge / release / Gerber joint with x [m].

## 2. Geometry & dimensions
A) Extract ALL lengths from the dimension chain (tick marks + numbers under the beam).
   Dimension line positions OVERRIDE guessed load positions when they conflict.
B) x=0 at the leftmost point on the beam axis; L = sum of ALL segments.
C) Units on paper:
   • meters (m) — use as-is
   • centimeters (cm) — convert to m (÷100)
   • kN / kN/m — convert to ton / ton/m (÷10)
D) Output JSON values in meters and ton only.

## 3. Loading system (only AFTER Phase A base identification is complete)
Scan left → right. One UDL span = one loads[] entry (x1, x2, w) — never duplicate.
**POSITION RULE (concentrated loads):** center on beam axis → vertical down to dimension line → that x [m] (inclined = where tip meets beam).
Distributed loads: use distributed_loads_protocol — start_x / end_x from dimension line only.
A) Concentrated forces: value [ton], arrow direction, position x [m] from dimension chain.
B) Distributed loads: output ONLY in distributed_loads[] — see distributed_loads_protocol (q, kN/m, arrows/block/wavy line).
C) External moments: M [ton·m], position x, rotation sense (counter-clockwise = positive in JSON).

## 5. Statics iron rules (from official formula sheet — mandatory)
- Inclined load / vector: decompose FIRST — Fx=F·cos(α), Fy=F·sin(α), before any equilibrium solve.
- Diagram jumps: N ← up, → down; M clockwise ↻ up, counter-clockwise ↺ down.
- Load→Q→M: concentrated→Q constant/M linear; UDL→Q linear/M parabolic.
- Area formulas: triangle b·h/2; trapezoid h(b1+b2)/2; unknown zero-shear triangle M_max=Q²/(2q).
